# Antigravity & GitHub Integration

이 문서는 Antigravity AI 에이전트가 GitHub 저장소와 어떻게 상호작용하고 연동되는지 설명합니다.

## 개요 (Overview)

Antigravity는 사용자의 로컬 개발 환경에서 동작하는 AI 코딩 어시스턴트입니다. GitHub와의 연동은 별도의 복잡한 설정 없이, 로컬 환경에 설치된 `git` 명령어를 통해 이루어집니다.

## 작동 방식 (How it works)

Antigravity는 터미널 명령어를 실행할 수 있는 권한을 가지고 있어, 개발자와 유사한 방식으로 Git 워크플로우를 수행할 수 있습니다.

1.  **코드 작성 및 수정**: Antigravity가 사용자의 요청에 따라 로컬 파일 시스템의 코드를 수정합니다.
2.  **Git 조작**: Antigravity는 `git status`, `git add`, `git commit`, `git push` 등의 명령어를 직접 실행하여 변경 사항을 관리합니다.
3.  **GitHub 동기화**: 로컬에서 커밋된 변경 사항을 원격 GitHub 저장소(Remote Repository)로 푸시하여 동기화합니다.

## 주요 기능

*   **자동 커밋**: 작업 단위별로 의미 있는 커밋 메시지와 함께 코드를 커밋할 수 있습니다.
*   **PR 생성 지원**: (GitHub CLI `gh`가 설치된 경우) 터미널을 통해 Pull Request를 생성하거나 관리할 수 있습니다.
*   **충돌 해결**: `git pull` 시 발생하는 간단한 병합 충돌을 분석하고 해결을 도울 수 있습니다.

## 요구 사항 (Prerequisites)

Antigravity가 GitHub와 원활하게 연동되기 위해서는 로컬 환경에 다음이 설정되어 있어야 합니다:

*   **Git 설치**: `git` 명령어를 사용할 수 있어야 합니다.
*   **인증 설정**: GitHub에 접근할 수 있는 SSH Key 또는 Credential Helper가 설정되어 있어야 합니다.
*   **저장소 연결**: 작업 중인 디렉토리가 Git 저장소로 초기화되어 있고(`git init`), 원격 저장소(`git remote add origin ...`)와 연결되어 있어야 합니다.
